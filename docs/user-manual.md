# Claudio FM 用户手册

**Claudio FM** 是一个智能 DJ 系统。你跟它聊天，它帮你选歌、播报、控制智能设备。

```
你说 "来点爵士" → Claude 推理 → 搜索网易云 → 播放 + TTS 播报
```

## 1. 快速开始

### 启动服务

```bash
cd Claudio
npm run dev          # 主服务 (端口 3005)
```

需要网易云 API：

```bash
cd api-enhanced
PORT=3001 node app.js   # NCM API (端口 3001)
```

### 浏览器打开

```
http://localhost:3005
```

### 首次配置

1. 点右上角 ⚙ 打开设置
2. 填 **API Key**（Anthropic 或 OpenAI-compatible key）
3. 可选：填 **Base URL**（默认 `https://api.anthropic.com`，用 DeepSeek 等改之）
4. 可选：点模型输入框旁的 **▼** 拉取可用模型，选一个
5. 点 **Test Connection** 验证连接
6. 点 **Save**

配置完即可在聊天框输入指令。

## 3. 聊天与选歌

这是最核心的功能。在底部输入框打字，Claude 会理解你的意图并推荐音乐。

### 示例

```
▸ "来点爵士"
▸ "上班路上提提神"
▸ "周杰伦的歌"
▸ "心情不好，来点治愈的"
▸ "适合下雨天听的"
▸ "今天有什么推荐的？"
```

### 系统会

1. 自动获取**天气、日历、时间**上下文
2. 调用 Claude 推理，理解你的情绪和需求
3. Claude 返回 JSON 格式的播放指令
4. 搜索网易云音乐并播放
5. 同时用 TTS 语音播报 DJ 文案

### 简单命令

以下命令**不经过 Claude**，直接执行：

```
下一首 / next
暂停 / pause
继续 / resume
上一首 / prev
音量加 / volume_up
音量减 / volume_down
stop / play / start
```

## 4. 播放模式

### 普通模式（Normal）

默认模式。你通过聊天输入指令，Claude 每次回复一组歌曲。播完后停止，等待你下一次指令。

状态栏显示曲目名称，无特殊标记。

### SMART 模式

点击 **SMART** 按钮启动。基于当前歌曲，调用网易云智能推荐算法，自动连续推送相似歌曲。

播完一首自动续下一首，状态栏显示 **SMART** 标记。

触发方式：
- 点 SMART 按钮
- 聊天发送指令后，Claude 返回的歌单播完会自动续推（标记 `smart: true`）

### FM 模式

点击 **FM** 按钮启动。调用网易云私人 FM 算法，随机推送你可能喜欢的歌。

播完一首自动续一首，状态栏显示 **FM** 标记。

### 模式对比

| 模式 | 谁选歌 | 自动续播 | 适用场景 |
|---|---|---|---|
| Normal | 你打字告诉 Claude | 否 | 有明确想听的 |
| SMART | 网易云算法 + Claude | 是 | 想听类似的但懒得想 |
| FM | 网易云私人 FM | 是 | 想探索新歌、随缘听 |

## 5. 场景建议

系统每 5 分钟检查一次上下文（天气、时间、日历），自动推送场景化音乐建议。

推送时会弹出 **Toast 卡片**，右上角 🔔 亮起。点击**播放**快速发送聊天指令。

### 可用场景

| 场景 | 触发条件 | 建议文案 |
|---|---|---|
| 🌧 雨夜 | 18-20点 + 下雨 | "下雨了，来点爵士？" |
| 🚗 早通勤 | 周一 7-9点 | "周一早上，提神节奏？" |
| 🍺 周五晚 | 周五 18-22点 | "周五晚上，放松一下？" |
| 🎂 生日 | 日历含"生日" | "今天有人生日，来点庆祝歌？" |
| 🌙 深夜 | 22点后或6点前 | "夜深了，轻柔助眠？" |
| ☀ 周末好天气 | 周末 + 晴天 10-14点 | "好天气，来点轻松的？" |

每条建议有 2 小时冷却，不会重复打扰。

## 6. 设置面板

点右上角 **⚙** 打开。

| 字段 | 说明 |
|---|---|

| 字段 | 说明 |

| 字段 | 说明 |
|---|---|

## 7. 网易云音乐登录

点导航栏 **LOGIN** 按钮打开登录弹窗。

### 二维码登录（推荐）

1. 弹窗默认显示二维码
2. 打开网易云音乐 App → 扫一扫
3. 扫码后自动确认登录

### 手机号登录

1. 点"手机号登录"切换
2. 输入手机号和密码
3. 点"登录"

登录后：
- 可搜索、播放高品质音乐（192k+）
- 可使用私人 FM、智能推荐
- 登录状态持久化，重启后保持

## 9. 用户语料系统

在 `user/` 目录下的文件用于个性化 Claude 的 DJ 行为。

| 文件 | 内容 | 示例 |
|---|---|---|
| `taste.md` | 音乐品味 | "喜欢爵士、后摇、电子" |
| `routines.md` | 日常作息 | "工作日 9 点上班，23 点睡觉" |
| `mood-rules.md` | 情绪→音乐映射 | "emo 的时候听平井大" |
| `playlists.json` | 预设歌单 | `{"工作专注": ["歌单链接"]}` |

这些文件的内容会在每次聊天请求时拼入 Claude 的 prompt，让推荐更懂你。

**目录路径**：通过设置面板的 `User Corpus Dir` 字段配置，默认 `user/`。

## 10. 架构说明

```
用户 / 前端 (自然语言指令)
    │
    ▼
┌──────────────────────────────────┐
│  Context Collector               │  ← 天气 + 日历 + 时间 + 历史 + 用户语料
└──────────────┬───────────────────┘
               │
               ▼
┌──────────────────────────────────┐
│  Claude / LLM (唯一大脑)          │  ← 推理 + 输出 JSON 动作序列
└──────────────┬───────────────────┘
               │
               ▼
┌──────────────────────────────────┐
│  Action Executor                 │  ← 解析 JSON → 搜索 + 播放 + TTS + UPnP
└──────────────┬───────────────────┘
               │
     ┌─────────┼─────────┐
     ▼         ▼         ▼
  NCM API   Fish TTS   UPnP 设备
```

### 关键设计原则

1. **Claude 是唯一大脑** — 所有音乐推荐、文案、转场由 LLM 决策，代码不做硬编码意图匹配
2. **上下文先行** — 调用 LLM 前自动收集天气、日历、历史、用户偏好，让决策充分知情
3. **纯 JSON 通信** — LLM 输出结构化的 `{ say, play, reason, segue, mood, arc }`，直接可执行
4. **实时推送** — 播放、TTS、Token 用量通过 WebSocket 推送到前端，无需轮询

### 技术栈

| 层 | 技术 |
|---|---|
| 运行时 | Node.js + TypeScript (tsx) |
| HTTP | Express 5 |
| 实时通信 | WebSocket (ws) |
| 数据库 | SQLite (better-sqlite3) |
| LLM | Anthropic Anthropic / OpenAI-compatible API |
| 前端 | 单文件 HTML/CSS/JS，无框架 |
| 音乐源 | 网易云音乐 (api-enhanced) |
